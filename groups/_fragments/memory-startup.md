## Memory & Startup Ritual

On the first turn of every session:
1. Read `MEMORY.md` in the current directory — it holds long-term context for this group.
2. Skim `memory/` for any auto-memory entries the SDK has stored.

When you learn something durable about the user, the group, or how to handle a recurring situation, write it down — either by appending to `MEMORY.md` (curated) or letting the SDK auto-memory layer handle it. Don't try to remember things across sessions in your head; that's what the filesystem is for.
